## Bootstrap (Task 1 execution)

Run from a working copy. These commands require network access (Composer/npm).

```bash
# 1. Generate the Laravel 11 framework skeleton in a temp dir, then overlay these files.
#    (Keeps the ICS overlay — composer.json, config/ics.php, .env.example, tailwind,
#     vite, postcss, pint, phpstan, phpunit, resources/, lang/ — authoritative.)
composer create-project laravel/laravel _skeleton "11.*"
#    Copy any skeleton-only files NOT already present here (artisan, bootstrap/,
#    public/index.php, default config/*.php, app/Http/Controllers/Controller.php,
#    app/Providers/AppServiceProvider.php) into the project root. Do NOT overwrite
#    the ICS overlay files listed above. Then remove _skeleton.

# 2. Install PHP dependencies from the ICS composer.json (Laravel + approved packages).
composer update

# 3. Enable the API layer (creates routes/api.php, wires Sanctum) — D-023.
php artisan install:api

# 4. Publish Spatie permission config + migration stubs — D-021.
php artisan vendor:publish --provider="Spatie\Permission\PermissionServiceProvider"

# 5. Frontend toolchain.
npm install

# 6. Environment + app key.
cp .env.example .env       # in production, place .env OUTSIDE the web root (D-039 SEC-02)
php artisan key:generate

# 7. Verify.
php artisan --version      # expect 11.x
npm run build
```

> Production deploy uses `composer install --no-dev --optimize-autoloader` and
> `npm run build`, with document root pointed at `/public` only (D-039 SEC-02).

## Environment profiles (D-037)

The ONLY difference between shared hosting and VPS is `.env`. See the shared-vs-VPS
delta table in [`.claude/SPRINT_1_TASK_1_PROJECT_SCAFFOLD.md`](.claude/SPRINT_1_TASK_1_PROJECT_SCAFFOLD.md).
No code or schema differs between environments. Migration to VPS is configuration only.

| Concern | Shared (Phase 1) | VPS (Phase 2) |
|---|---|---|
| Session / Cache | file | redis |
| Queue | database (cron) | redis (Horizon) |
| Feature flags (`config/ics.php`) | all false | enabled as needed |

## Testing

PHPUnit 11 (Laravel 11 default). Unit/feature suites run on SQLite `:memory:` for
speed. Engine-specific behaviour (JSON, FULLTEXT) is validated in CI against the
**pinned production engine** (MySQL 8 / MariaDB — LIM-03 / M-3); Sprint 1 core
tables use no engine-specific features.

```bash
php artisan test
./vendor/bin/pint --test     # code style
./vendor/bin/phpstan analyse # static analysis
```
